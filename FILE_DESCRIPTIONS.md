# Project File Overview

This document describes the purpose and behavior of each file in this repository.

## collect.py
- Main batch collection pipeline for MOLIT apartment trade data.
- Loads secrets from `.env` (MOLIT_API_KEY, SUPABASE_URL, SUPABASE_KEY).
- Fetches each region’s `LAWD_CD` for the past 3 months via Public API.
- Parses XML transactions, normalizes values, and deduplicates.
- Upserts the data into Supabase `properties` and `transactions` tables.
- Includes robust error handling:
  - XML parse failure fallback
  - invalid numeric fields skip items
  - missing environment variables fail early

## api/main.py
- FastAPI service exposing `/listings` endpoint.
- Reads `transactions` table joined with `properties` table from Supabase.
- Supports query filters: `region_code`, `dong`, `min_area`, `max_area`, `limit`.
- Ensures empty results return `[]` and `count: 0`.

## requirements.txt
- Lists runtime dependencies:
  - `supabase`
  - `python-dotenv`
  - `requests`
  - `fastapi`
  - `uvicorn`

## .env (not committed)
- Expected variables:
  - `MOLIT_API_KEY`
  - `SUPABASE_URL`
  - `SUPABASE_KEY`

## __pycache__/
- Bytecode cache generated by Python.
- Not source-controlled, safe to ignore.
